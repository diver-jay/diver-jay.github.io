# 2단계: 초안 작성 (Drafting) 세부 프로세스

빈 화면의 공포를 없애고, '데이터 구조를 잡고 살을 붙여나가는' 엔지니어링 방식으로 접근하는 초안 작성 5단계입니다.

## 2-1. 뼈대 확장 (Keyword & Punchline Mapping)
미리 잡아둔 5단계 흐름(도입-반전-전개-절정-결론)에 뼈대를 세우는 작업입니다.
* **작업 내용:** 각 단계를 관통하는 **핵심 키워드**와 **핵심 문장(펀치라인)**을 1~2개씩만 작성합니다.
* **예시 ([2] 반전):**
  * **키워드:** 슬랙 P0 알림, 식은땀, 롤백 고민
  * **펀치라인:** "배포 성공의 안도감이 가시기도 전, 슬랙의 빨간 알림 배지가 모든 것을 바꿔놓았다."

## 2-2. 기술적 팩트 덤프 (Technical Fact Dump)
스토리텔링이나 감정 묘사는 잠시 미뤄두고, 글의 코어가 되는 기술적 사실만 먼저 건조하게 나열합니다.
* **작업 내용:** **[3] 전개(원인)**와 **[4] 절정(해결)** 부분의 팩트를 불릿 포인트나 의사코드(Pseudocode) 수준으로 거침없이 쏟아냅니다.
* **체크 포인트:** * 어떤 로직/쿼리에서 문제가 발생했는가?
  * 핫픽스는 구체적으로 어떤 절차로 진행했는가?

## 2-3. 서사 및 브릿지 연결 (Narrative Bridging)
작성해둔 기술적 팩트 앞뒤로 스토리텔링 요소를 붙이고, 각 단계를 매끄럽게 연결합니다.
* **작업 내용:** **[1] 도입, [2] 반전, [5] 결론**의 내용을 채우고, 시각 자료가 들어갈 자리를 표시(Placeholder)합니다.
* **메모 예시:** * `[여기에 슬랙 에러 로그 캡처 이미지 삽입]`
  * `[이 구간부터 심각한 어투로 전환]`
  * `[인증 플로우 아키텍처 다이어그램 들어갈 자리]`

## 2-4. 논스톱 타이핑 (Continuous Brain Dump)
앞서 만들어둔 뼈대, 팩트, 메모를 바탕으로 처음부터 끝까지 멈추지 않고 줄글을 써 내려갑니다.
* **작업 내용:** 문맥이 어색하거나 단어가 떠오르지 않아도 **절대 뒤로 가기(Backspace)를 누르지 않고** 흐름을 타서 끝까지 작성합니다.
* **팁:** 막히는 부분은 `[여기에 더 좋은 표현 찾기]`, `[정확한 로그 기록 확인 후 수정]` 등 대괄호 마커만 남기고 넘어갑니다.

## 2-5. 1차 밸런스 체크 (Structural Balance Check)
완성된 거친 초안을 처음부터 끝까지 빠르게 읽어보며 글의 구조적 비율을 점검합니다.
* **작업 내용:** 기술 블로그의 본질과 독자의 몰입도 사이의 밸런스를 확인합니다.
* **체크 포인트:**
  * 기술적 설명([3] 전개)이 너무 길어 독자의 텐션이 떨어지지 않는가?
  * 감정 묘사가 너무 길어 본질을 흐리지는 않는가? (불필요한 문단 덜어낼 준비)
