## 메타 정보

| 항목 | 값 |
|---|---|
| 시리즈 | "그대들 어떻게 살 것인가" 2편 중 1편 |
| 제목(가제) | 그대들 어떻게 살 것인가: AI 시대, 하네스 프로그래밍으로 3D의 두려움을 깨다 |
| 핵심 메시지 | 진짜로 대체 가능한 것인가? 직접 해보니 두려움이 clear해졌다 |

## 시리즈 구조

| 편 | 제목 | 핵심 메시지 |
|---|---|---|
| **1편 (본 글)** | 하네스 프로그래밍 3D 도전기 | 직접 해보면 두려움이 clear해진다 |
| 2편 | "우리 프로그래머들"을 읽고 | 개발자는 사라지지 않는다 |

## 확정 사항

### 톤/문체: 혼합 (D)
- **도입/결론** → 에세이체: 권위와 인사이트 전달 (4장: 권위 포장)
- **전개 1~2** → 실황체: 동질감/몰입 극대화 (3장: 감정 몰입)
- 톤 전환 자체가 감정 낙폭을 만드는 장치로 활용

### 하네스 프로그래밍 용어 포지셔닝: (C) 인용 + 확장
- 기존에 정의된 용어를 인용하되, 본인의 3D 도전 체험으로 의미를 확장

### 도입 장면: (A+B) 혼합
- 구체적 AI 뉴스/트윗 장면 + 동료와의 대화를 조합
- 다크심리학 3장 "생생하게 묘사하라" + "나와 똑같다고 느끼게 하라" 적용

### 도구명 언급: (C) 서사 속 녹이기
- Claude Code / feature-dev를 서사 흐름에 자연스럽게 삽입
- 4장 "과시가 아닌 자연스러운 노출"에 부합

### 핵심 에피소드: 해파리 3D 모델링
1. Google에서 해파리 사진 뽑아서 "이 모양으로 3D 모델링 해줘" → 결과물 허접 → "그거 말고"의 반복
   - 깨달음: 3D 모델링 용어/스킬/패턴을 모르니 지시 자체가 모호
2. WebGL 코드 리뷰 불가 → 피드백도 모호 ("너무 물 색깔이 아니야" 수준)
   - 결론: 공부하면서 질문/피드백 수준이 점진적으로 향상

### Middle-out 인사이트: 1편에서 완결
- Bottom-up도 Top-down도 아닌, 중간에서 위아래로 확장하는 학습법
- 1편 결론부에서 도출
- **홍정모 채널 레퍼런스** ("AI 시대 공부 순서, 대부분이 틀리고 있습니다")
  - Top-down의 함정: 겉핥기만 하고 끝남 → AI의 소비자로 전락
  - Bottom-up의 함정: 토끼굴에 빠짐 → 금방 지침
  - 본인 경험과의 차이: 이미 전문성이 있는 중간 지점(React)에서 시작 → 필요에 의해 위아래로 확장
  - 핵심: "필요에 의한 학습" — 부딪혀서 필요해졌을 때 찾아봄 (Bottom-up 아님), 땜빵에 만족하지 않고 근본 원인을 팜 (Top-down 아님)

### 결과물 공개: GitHub 링크 + GIF
- Vercel 배포는 추후 고려

## 글 구조 (4섹션)

### 도입부 (~300자) — 에세이체 + 실황 장면
- AI 뉴스/동료 대화 장면으로 두려움 환기
- "직접 해보기로 했다" 선언
- **다크심리학:** 희망과 두려움 동시 자극(3장), 나와 똑같다고 느끼게 하라(3장), 영웅 서사 구조(2장)

### 전개 1 (~400자) — 에세이체 → 실황체 전환
- 극단적 선택지 2개 제시: 바닥부터(WebGL) vs 안전지대(React만)
- 중간 선택: r3f + 하네스 프로그래밍
- **다크심리학:** 중간 선택지를 매력적으로 포장하라(2장) — 미끼 효과(Decoy Effect)

### 전개 2 (~600자, 글의 핵심) — 실황체
- 해파리 에피소드: "그거 말고"의 반복 → 용어를 모르니 지시가 모호
- WebGL 리뷰 불가: 모호한 피드백밖에 줄 수 없음
- 깨달음: Trash in, Trash out. 공부할수록 질문 수준이 올라감
- **다크심리학:** 약점과 실수를 공개하라(4장), 자기비하로 다가가라(3장)

### 결론 (~400자) — 에세이체
- Middle-out 학습법 인사이트 도출
- GitHub 링크 + 결과물 GIF
- 수미상관: 도입의 두려움 → "앞으로의 기술 변화가 선명하게 기대된다"
- **다크심리학:** 전문 용어와 수치로 권위 구성(4장), 결과물로 논리(Logos) 증명

```
Step 1. 팩트 덤프 — GitHub 커밋 기반 기술적 사실 나열 (2-2)

```

## Step 1 결과: 팩트 덤프

### 프로젝트 개요
- **기간**: 2/8 ~ 3/15 (약 5주), 총 47 커밋
- **레포**: https://github.com/diver-jay/r3f-drei-water-caustics-effect

### Phase별 타임라인

**Phase 1: 기반 구축 (2/8-10)** — React 패턴으로 이해 가능한 영역
- 프로젝트 초기화 + Water Caustics Effect 구현, JS→TS 마이그레이션
- Provider 패턴 적용, Water Surface/Pool 구현
- `docs/pool-rendering-issues.md` 160줄 TIL 작성 (Depth Buffer, Transmission 렌더 패스, Fresnel)

**Phase 2: 시각 효과 확장 (2/17)**
- Rising Bubbles 추가, 색상 RED/YELLOW/GREEN

**Phase 3: 해파리 도전 (2/20-23)** — 핵심 에피소드
- `d457e82` 해파리 첫 추가: `Jellyfish.jsx` **1,145줄** + GLSL 셰이더 11개 + `wanted.png` (Google 해파리 사진)
- 같은 날 5마리 추가 → Impulse velocity → 애니메이션 분화
- `6cd9f83` Evan Wallace 영감 — water cyan color + IOR 굴절 + refracted light (셰이더 주석: `// Reference: madebyevan.com/webgl-water`)
- **버그 3연타**:
  - `27904a5` Parallel Transport Twist — `setFromUnitVectors` → 직교 기저벡터 3축 직접 계산 + `makeBasis`
  - `2302f5a` Max Speed — 단순 클램핑 (`MAX_SPEED = 3.0`)으로 땜빵
  - `c32124d` Verlet velocity explosion — `Math.pow(0.82, dt)` damping + world→model 좌표 변환 (전날 `MAX_SPEED` 삭제)

**Phase 4: 인터랙션 & 폴리싱 (3/7-15)**
- Gazing mechanism + 파티클 시스템, 해파리 크기별 물리 반응 차별화
- Water drop effects, Memory leak 수정 (material/geometry/effect dispose 일괄)

### 블로그 핵심 팩트 5개
1. `wanted.png` — "이 모양으로 만들어줘" 지시의 증거
2. 버그 3연타 타임라인 — MAX_SPEED 땜빵(2/21) → Verlet damping 근본해결(2/22)
3. TIL 문서 — "왜 안 보여?" → "Transmission 렌더 타겟에서 transparent 제외" 수준으로 질문 레벨업
4. Evan Wallace 레퍼런스 — 공부하니까 검색어가 달라지고 레퍼런스를 읽을 수 있게 됨
5. 학습 곡선: 초기 Add → 중반 Bug fix → 후반 Implement mechanism + dispose 관리

## Step 2 결과: 뼈대 확장

### 도입부 (~300자) — 에세이체 + 실황 장면
- **키워드**: AI 대체 공포, 동료 대화, 직접 해보기 선언
- **펀치라인**: "대체 가능한지 묻기 전에, 대체해 보기로 했다."
- **다크심리학**: 희망과 두려움 동시 자극(3장), 나와 똑같다고 느끼게 하라(3장)

### 전개 1 (~400자) — 에세이체 → 실황체 전환
- **키워드**: Top-down/Bottom-up 함정, 중간 선택, r3f + 하네스 프로그래밍
- **펀치라인**: "React는 알았다. WebGL은 몰랐다. 그런데 r3f라는 게 있었다."
- **다크심리학**: 미끼 효과 — 극단적 선택지 2개(겉핥기 vs 토끼굴) + 중간 선택지
- **복선**: Middle-out의 시작점 = 이미 아는 React

### 전개 2 (~600자, 핵심) — 실황체
- **키워드**: Trash in Trash out, "그거 말고", 용어 부재, 질문 수준 상승
- **펀치라인**: "'해파리가 이상하게 돌아' — Parallel Transport를 모를 때 줄 수 있는 최선의 피드백이었다."
- **팩트 배치**: wanted.png → 1,145줄 → "그거 말고" → Parallel Transport → MAX_SPEED 땜빵 → Verlet damping
- **다크심리학**: 약점과 실수 공개(4장), 자기비하(3장)
- **GIF**: 해파리 초기 → 최종 비교

### 결론 (~400자) — 에세이체
- **키워드**: Middle-out, 필요에 의한 학습, 두려움 → 기대
- **펀치라인**: "내가 이미 아는 것이 고삐가 되고, 모르는 것이 방향이 됐다."
- **다크심리학**: 전문 용어와 수치로 권위(4장), 결과물로 Logos 증명
- **GIF**: 완성된 Water Caustics 인터랙션
- **수미상관**: 도입의 두려움 → "앞으로의 기술 변화가 선명하게 기대된다"

### 자수 배분: ~1,700자 (목표 1,500~2,300 내)

## Step 3 결과: 서사 브릿지

### 설득 구조: 영웅 서사 (2장) + 다크심리학 레이어

| 섹션 | 영웅 서사 | 다크심리학 레이어 |
|---|---|---|
| 도입 | **위기** — AI 대체 두려움 | 3장: 나와 똑같다고 느끼게 하라 + 희망과 두려움 동시 자극 |
| 전개 1 | **고난 진입** — 선택의 기로 | 2장: 미끼 효과 (Top-down/Bottom-up/Middle-out) |
| 전개 2 | **고난 심화** — "그거 말고", 버그 3연타 | 3장: 자기비하 + 4장: 약점과 실수 공개 |
| 결론 | **극복** — Middle-out, 결과물 | 4장: 전문 용어와 수치로 권위 + Logos 증명 |

### 서사 브릿지 상세

**도입부** `[톤: 에세이체, 약간의 무게감]`
- 장면: 점심시간 동료 대화. "프로그래머 다 대체된다며" 류의 발언. 같은 두려움.
- 브릿지 → 선언: 두려움의 정체를 알고 싶었다. 직접 해보기로 했다. AI한테 내가 모르는 걸 시키면 어떻게 되는지.

**전개 1** `[톤 전환 지점: "그래서 터미널을 열었다" → 실황체로]`
- 브릿지 in: 선택지 제시 (미끼 효과)
  - WebGL 바닥부터 → 토끼굴, 해파리까지 도달 전에 지침
  - React 안에서만 → 겉핥기, AI의 소비자로 남음
- 중간 선택: React는 알았다. WebGL은 몰랐다. r3f가 있었다. 이미 아는 것을 고삐(하네스)로 쓰고 AI에게 모르는 영역을 시키기로.

**전개 2** `[톤: 실황체, 자기비하 유머]`
- 에피소드 1 "그거 말고": Google 해파리 사진 → "이 모양으로" → 1,145줄 + 셰이더 11개 → 맞는지 틀린지 판단 불가 → "좀 더 자연스럽게" "그거 말고"
- 에피소드 2 버그 3연타: "이상하게 돌아" → Parallel Transport / MAX_SPEED 땜빵 → 다음 날 Verlet damping 근본 원인
- `[GIF: 해파리 초기 → 최종 비교]`
- 브릿지 out: Trash in, Trash out. 공부할수록 질문이 정확해지고, 정확한 질문이 정확한 결과를 만들었다.

**결론** `[톤: 에세이체, 도입의 두려움을 뒤집는 확신]`
- Middle-out: 내가 이미 아는 것이 고삐가 되고, 모르는 것이 방향이 됐다.
- `[GIF: 완성된 Water Caustics + 해파리 인터랙션]` + GitHub 링크
- 수미상관: 두려움은 모호함에서 왔다. 직접 해보니 모호함이 걷혔다. 앞으로의 기술 변화가 선명하게 기대된다.
